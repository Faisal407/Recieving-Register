☁ Cloud Sync — Apps Script Setup

One-time setup, reusable for every project. Uses a free Google Apps Script Web App — no OAuth login needed in this page, since the script runs under your own Google account's authorization once deployed. Each project links its own Sheet ID + Web App URL from its card in the Projects tab. This version also lets a second device read back what's already in the Sheet (not just write to it) — that's what powers "Join via Code" and "Sync from Sheet".

1. Open (or create) the Google Sheet for a project.
2. Extensions → Apps Script.
3. Delete any starter code, paste the script below.
4. Replace PASTE_YOUR_SHEET_ID_HERE with that Sheet's ID (from its URL, between /d/ and /edit).
5. Deploy → New deployment → Type: Web app → Execute as: Me → Who has access: Anyone.
6. Copy the URL (ends in /exec) into that project's card on the Projects tab, then use Copy Share Code there to send it to other devices.

Known limitation: writes (Save Entry, dispatcher upload) go through doPost, and the browser still can't read that response — it marks "Sent (unconfirmed)". Reads (Join via Code, Sync from Sheet) go through doGet via a JSONP trick, which does work, so pulling existing records is reliable — writing just isn't confirmable. Keep exporting Excel per project as backup regardless. OneDrive auto-export has no equivalent no-login trick — that genuinely needs the Phase-2 backend with Microsoft sign-in.
